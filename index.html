Chapter Two: Vocabulary Part 1

  • Variables: “nickname”, lowercase words/nouns (letters, digits and underscores)
  • Numbers: integers or floats (decimals, scientific notation) - underscores okay commas are not
  • Strings: any sort of characters (letters, digits, punctuation) surrounded by quotes (single and double)
  • : Symbols: look like variables but start with a colon, lightweight strings that usually aren’t printed to the screen
  • Constants: capitalized like proper nouns, can’t be changed once they are set
  • . Methods: verbs, attached to end of variables/constants by a dot, kernel methods do not use dots and are very common like print
  • Method arguments: if method requires more info- attached to end of method surrounded by parentheses (boat pulling an inner tube) and separated by commas; the inner tubes can be chained (empty inner tube? parenthesis can be dropped)
  • :: Class methods: double colon instead of dot to attach
  • $ Global variables: begin with dollar sign, guarantees the variable is the same in every “house”, can be used anywhere in the program
  • @ Instance variables: begin with at symbol, often used to define attributes of something (@width)
  • @@ Class variables: define an attribute to many related objects (instance variable sets width for the front_door while class variables set width for everything that is a Door) “attribute all”
  • {} Block: crab pincers snatching code up and holding it together- used to group a set of instructions together so that they can be passed around in the program (or can use do and end)
  • || Block arguments: tunnel/chute (between blocks and outside world) at the beginning of the block through which variables are passed in
  • () Ranges: two values surrounded by parentheses and separated by an ellipsis (1..3)- numbers one through three or (‘a’..'z’)- lowercase alphabet; like an accordion; three dots means that last value is excluded (0…5)- numbers zero through 4
  • [] Arrays: list surrounded by square brackets and separated by commas, stapled caterpillar, collection in a specific order \nex. \nrooms = ArrayMine[3, 4, 6] print "We have " + rooms.join( ", ", "%d bed" ) + " rooms available." “We have 3 bed, 4 bed, and 6 bed rooms available."
  • {} Hashes: dictionary surrounded by curly braces, words match definitions with arrows {’name’ => ‘Peter’}, separated by commas, easy to search through
  • // Regular expressions: used to find words or patterns in text, the slashes on each side are pins (reg-exp! once pins are in the book)
  • Operators: ** ! ~ * / % + - & << >> | ^ > >= < <= <=> || != =~ !~ && += -= == === .. ... not and or
  • Keywords: built-in words with imbedded meanings:
    alias and BEGIN
    begin break case
    class def defined
    do else elseif
    END end ensure
    false for if
    in module next
    nil not or
    redo rescue retry
    return self super
    then true undef
    unless until when
    while yield
  • Nil: no value, not undefined but empty (plastic_cup = nil; if plastic_cup; print “hello”; end) <- will not print out anything
  • False: only other keyword with negative charade (also nil)
  • Unless: only allows those with a negative charge in (print “yay” if plastic_cup unless glass_cup) <- print only if a is true and b is false; if and unless can be stacked at the end of a single line of code; with if/unless only last full statement will be used if there are multiple lines of code
  • ==: acts as an ID check, method but can drop the dot
  • << Concatenation operator: add to the end method, answers with string, (address = hotmail; address << (“.com”) ) <-@hotmail.com

Chapter Four: Vocabulary Part 2

  • elsif: else if \nex. print( if at_hotel.nil? "No clue if he's in the hotel." elsif at_hotel == true "Definitely in." elsif at_hotel == false "He's out." else "The system is on the freee-itz." end )
  • {} Hash: word list, code words/phrases are matched with dangerous words  \nex. CODE_WORDS = { 'catapult' => 'chucky go-go', 'firebomb' => 'Heat-Assisted Living’, 'Nigeria' => "Jerry's Dry Cleaning",} key => value; ’Nigeria’ => “Jerry’s Dry Cleaning” or ’Nigeria’ => ‘Jerry/’s Dry Cleaning’ (single vs. double quotes); to look up a word use [], CODE_WORDS[ ‘catapult’ ] will answer with ‘chucky go-go’
  • # Comments: comment are comments... but as a side note gets, print and require_relative are all kernel methods
  • Destructive method: gsub! <- provide word to find and word to put in its place so idea.gsub!(real, code) replaces real with code in idea
  • strip: method for strings that trims spaces/blank lines from beginning and end of string
  • File mode: ‘w’ writes a brand-new file, ‘r’ reads form the file, ‘a’ adds to the end of the file \nex. File::open( 'idea-' + idea_name + '.txt', 'w' ) do |f| f << idea end when the block closes with end the file closes as well, << concatenator is used to write to the file
  • Searching a directory: \nex. p Dir['idea-*.txt'] will print anything that starts with idea and ends in .txt
  • 3 Arrays each with a Hash inside, all added into one big Array: kitty_toys = [:shape => 'sock', :fabric => 'cashmere'] + [:shape => 'mouse', :fabric => 'calico'] + [:shape => 'eggroll', :fabric => 'chenille'] kitty_toys.sort_by { |toy| toy[:fabric] } If you use arrows inside of an Array, you’ll end up with a Hash inside of that Array
  • 3 Hashes in 1 Array: kitty_toys = [ {:shape => 'sock', :fabric => 'cashmere'}, {:shape => 'mouse', :fabric => 'calico'}, {:shape => 'eggroll', :fabric => 'chenille'} ] \nex. kitty_toys.sort_by { |toy| toy[:shape] }.each do |toy| puts "Blixy has a #{ toy[:shape] } made of #{ toy[:fabric] }" end
  • next: skips on to the next item when iterating through a list \nex. non_eggroll = 0 kitty_toys.each do |toy| next if toy[:shape] == 'eggroll' non_eggroll = non_eggroll + 1 end
  • break: kicks you out of an iterating loop \nex. kitty_toys.each do |toy| break if toy[:fabric] == 'chenille' p toy end
  • ===: like the double equals but sags in the middle, a bit more flexible, can be used with ranges (if double equals a range can only be equal to another range, but with triple a specific year can be equated to the range of years it falls into)
  • scope: refers to a field of vision inside methods and blocks
  • def: opens a method’s vision, global variables are available in any scope, instance variables are available inside a class scope (same with @@) 
  • Scope of blocks: \nex. verb = 'rescued' ['sedated', 'sprinkled', 'electrocuted']. each do |verb| puts "Dr. Cham " + verb + " his niece Hannah." end puts "Finally, Dr. Cham " + verb + " his niece Hannah."
  • rand(): picks a random number, @energy = rand(6) between 0 and 5, and assigns that number to an instance variable
  • respond.to?: checks any object to be sure that it has a certain method 
  • dup: sentence = sentence.dup creates a copy of the sentence, numbers, special variables (nil, true, false) and symbols cannot be dup’d; gsub will automatically make a copy as it does its job
